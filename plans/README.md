  columns; the key always exists (worst case value is None, cosmetic). Not a crash.

  duration (get_db yields inside the lock), so API-vs-API interleaving can't happen; only an
  external CLI write could race, and the blast radius is a stale kanban card. Not worth doing.
- **Frontmatter `---`-in-prose truncation** — behavior matches the documented fence design
  (commit `0957c8d`: only bare `---` lines are fences; the first one after the opener closes
  the block). The residual edge (a bare `---` line inside a quoted YAML multiline string) is
  exotic; revisit only if a real criteria file hits it.

  load-bearing: it refreshes the overview strip and the NeedsAction rail after a state
  change, and a localhost fetch of a few hundred rows is negligible. By design.
- **Per-request YAML loads in rare endpoints** (`/api/portfolio/research`, `/api/onboarding`,
  `/api/cv/audit`) — micro-optimization at single-user scale (~ms per call, endpoints hit
  occasionally); caching would add stale-config risk for zero felt latency. Not worth doing.
- **`lib/index.ts` vs `lib/utils.ts` split** — cosmetic organization nit; `index.ts` is the
  facade, `utils.ts` holds `cn`. No drift, no duplication. Not worth doing.
- **`dashboard/backend/main.py` god module (801 lines)** — real but LOW-leverage right now:
  an APIRouter split is a multi-day mechanical refactor with regression risk and no feature
  payoff; revisit when the file next needs a structural change anyway. (Recorded, not planned.)
- **Security & dependency sweep** — clean: profile-id regex guards path construction,
