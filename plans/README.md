## Findings considered and rejected (so they aren't re-audited)

- **esbuild npm-audit (3 high, via Vite)** — dev-only, non-exploitable, and unpatchable until the
  registry advances past esbuild 0.28.1. Already documented in `docs/SECURITY.md`. SETTLED.
- **FastAPI no-auth binding to `127.0.0.1`** — intentional for a single-user local tool
  (documented in `dashboard/backend/main.py`). Not a finding (origin hardening is the optional
  plan 020, not an auth gap).
- **Broad `except Exception` in per-source/per-job loops** (`discovery/runner.py`,
  `run_brain.py`) — intentional isolation so one bad source/job can't empty the run; each logs to
  `source_health`/`events`. Not a finding.
- **ATS parsers (greenhouse/lever/ashby/smartrecruiters) "duplication"** — each maps a genuinely
  distinct API contract; the per-source code is near-irreducible field mapping, not shared logic.
  No abstraction warranted.
- **EN/ES outreach template pair** — parallel by design (two languages), not accidental dup.
- **`compute_job_id` truncated sha1 (16 hex)** collision risk — acceptable for a personal-scale
  job table; not worth widening.
- **CLI lazy imports inside typer commands** — intentional fast-startup convention.
- **`reportlab` Paragraph markup injection** — `render._esc` escapes `& < >` for every
  user-data Paragraph; the only literal `<b>` wrappers wrap already-escaped content. No live sink.
- **No-fabrication CV guarantee** — verified holding in `cv/tailor.py` (skills only from the
  user's real list; highlights filtered, never invented). No finding.

## How these were produced

Recon read README/docs/pyproject/schema and the full engine + dashboard surface first-hand.
Phase-2 audit fanned out 6 read-only category agents (correctness, security, performance,
tests-dx, tech-debt/deps, docs-direction) against the `improve` audit playbook. Every finding
that became a plan was re-opened and confirmed against the live code before planning (Phase 3),
and the code excerpts in each plan come from the advisor's own reads.
